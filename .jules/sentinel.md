## 2025-02-28 - Missing Application-Level Authorization Checks in Repositories
**Vulnerability:** IDOR (Insecure Direct Object Reference) vulnerabilities were present in the `delete` methods of `SupabaseTransactionRepository`, `SupabaseCardRepository`, and `SupabaseAccountRepository`. The `delete` methods only required an `id` parameter and didn't verify if the resource being deleted belonged to the currently authenticated user (`userId`).
**Learning:** Always include authorization checks at the lowest possible layer (e.g., repository or database) to enforce multi-tenant isolation. Merely relying on ID constraints without user validation leaves the system susceptible to unauthorized resource deletion if attackers can guess or brute-force valid resource IDs.
**Prevention:** Update repository interfaces (`delete(userId: string, id: string)`) to explicitly require `userId` and enforce `.eq('user_id', userId)` on deletion queries. Standardize this pattern across all repositories that deal with user-specific data.
